STEP 4 — FIX USER DATA ISOLATION

This is critical.

Every user-specific record must be connected to the authenticated Supabase user ID.

Use:

auth.users.id

as the identity source.

For orders, store:

user_id UUID NOT NULL

The user_id must reference the authenticated user.

When fetching orders, NEVER fetch every order.

Incorrect:

supabase
  .from('orders')
  .select('*')

Correct logic:

const {
  data: { user }
} = await supabase.auth.getUser();

if (!user) {
  return;
}

const { data, error } = await supabase
  .from('orders')
  .select(`
    *,
    order_items(*)
  `)
  .eq('user_id', user.id)
  .order('created_at', { ascending: false });

Only show orders belonging to the currently authenticated user.

Apply the same user ownership logic to:

cart_items
wishlist_items
addresses
notifications
profiles
